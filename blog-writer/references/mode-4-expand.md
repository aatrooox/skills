# 模式四：扩展模式

用户写核心观点，你来扩展补充。

## 触发条件

- 用户说"扩展一下"、"帮我补充"、"凑到1000字"

---

## 核心特点

- ❌ **不进行写作点评**：此模式不评价原文，只确保扩展部分风格一致
- 准确把握用户核心观点
- 严格遵循用户语气（参考 [user-profile.md](user-profile.md)）
- 不偏离核心观点

---

## 工作流程

### 1. 理解核心

准确把握用户想表达的核心观点。

**理解步骤**：

#### 识别核心论点
用户的中心观点是什么？

**示例**：
```
用户原文："稳定就是最大的牢笼。"

核心论点：稳定会限制成长
```

#### 识别支撑论据
用户已经给出了哪些支撑？

**示例**：
```
用户原文："我在公司待了3年，技能栈没变化。"

已有论据：个人经历（3年无成长）
```

#### 识别缺失部分
用户观点还缺少什么？

**示例**：
```
缺失部分：
- 更多案例支撑
- 反例讨论（为什么有人觉得稳定好？）
- 具体建议（如何打破稳定？）
```

---

### 2. 保持声音

扩展内容必须严格遵循 [user-profile.md](user-profile.md) 的"语气模仿指南"。

**查阅语气指南**：
扩展前必须查阅 user-profile.md，确保：
- 口语化表达
- 短句落点
- 反转对比
- 轻度自嘲（如适用）

**示例对比**：
```markdown
❌ AI 扩展腔：
"从长远的角度来看，我们应当重视个人成长，积极寻求突破，避免陷入舒适区。"

✅ 用户语气扩展：
"3年不成长，等于慢性自杀。看起来很安全，其实是在一点点失去竞争力。"
```

---

### 3. 补充论据

增加支撑观点的例子、数据、引用。

**补充类型**：

#### 类型一：更多案例
横向扩展，增加不同场景的例子。

**示例**：
```markdown
用户原文："我在公司待了3年，技能栈没变化。"

扩展补充：
"不只是我。我身边好几个同事也是这样。用的还是3年前的技术栈，项目还是3年前的老项目。看起来工作稳定，其实已经跟不上行业了。"
```

#### 类型二：具体数据
增加可量化的支撑。

**示例**：
```markdown
用户原文："感觉自己在慢性自杀。"

扩展补充：
"每年技术栈更新 20%，3年就是 60%。如果你 3 年技能不变，等于落后了 60%。市场不会等你。"
```

#### 类型三：反驳段
增加"有人可能会说..."的反驳段，增强说服力。

**示例**：
```markdown
扩展补充：
"有人会说，稳定本身就是一种竞争力，尤其是在经济下行的时候。

但这是短视的。短期看,稳定是优势；长期看，稳定是负债。3年不成长，就会被淘汰。"
```

#### 类型四：类比
用类比加深理解。

**示例**：
```markdown
扩展补充：
"稳定就像温水煮青蛙。水温一点点升高，青蛙感觉不到危险，等反应过来已经来不及了。"
```

#### 类型五：故事或案例
增加一个简短的故事。

**示例**：
```markdown
扩展补充：
"我认识一个前同事，在一家国企待了 5 年。工作轻松，收入稳定，看起来很爽。

去年被裁了。出来找工作才发现，市场早就不需要他那套技能了。面试了十几家，一个 offer 都没拿到。

5年稳定，换来的是完全失去竞争力。"
```

---

### 4. 控制篇幅

达到目标字数（通常1000字左右）。

**篇幅控制技巧**：

#### 计算当前字数
```bash
# 统计中文字符数（不含空格和标点）
echo "【文本内容】" | grep -o '[一-龥]' | wc -l
```

#### 分配扩展比例
```
原文：200 字
目标：1000 字
需扩展：800 字

分配：
- 补充案例：300 字
- 反驳段：200 字
- 类比：100 字
- 故事：200 字
```

#### 避免过度扩展
- 不要为了凑字数而重复
- 不要偏离核心观点
- 保持节奏感（不要全是密集信息）

---

### 5. 不进行写作点评

此模式下，**只确保**：
- ✅ 扩展部分符合用户语气（查阅 user-profile.md）
- ✅ 不偏离核心观点
- ✅ 达到目标字数

**不做**：
- ❌ 评价原文质量
- ❌ 指出原文的修辞问题
- ❌ 批评用户的表达方式

---

## 扩展技巧详解

### 技巧一：分层展开

把观点拆分成多个层次。

**示例**：
```markdown
用户原文："稳定就是最大的牢笼。"

分层展开：
1. 为什么是牢笼？（失去竞争力）
2. 牢笼如何形成的？（舒适区 → 惰性 → 停止成长）
3. 如何打破牢笼？（主动制造不适感）
```

---

### 技巧二：多维度分析

从不同角度看同一个观点。

**维度示例**：
- 个人维度：对个人成长的影响
- 市场维度：市场竞争力变化
- 时间维度：短期 vs 长期
- 场景维度：不同行业/岗位的差异

**示例**：
```markdown
用户原文："稳定会限制成长。"

多维度扩展：

**从个人角度**：技能停滞，思维固化。
**从市场角度**：行业更新快，稳定 3 年等于落后 60%。
**从时间角度**：短期看是优势，长期看是负债。
**从场景角度**：国企稳定 vs 互联网快速迭代。
```

---

### 技巧三：对比强化

用对比增强观点的表达力。

**对比类型**：
- 现在 vs 未来
- 表面 vs 实际
- 你 vs 别人
- A 选择 vs B 选择

**示例**：
```markdown
用户原文："稳定看起来安全，其实危险。"

对比扩展：
"看起来：工作稳定，收入稳定，生活稳定。
实际上：技能停滞，竞争力下降，未来堪忧。

看起来在保护自己，实际上在消耗自己。"
```

---

### 技巧四：引入外部声音

引用别人的观点或数据支撑。

**引用来源**：
- 行业报告数据
- 名人观点
- 研究结论
- 他人经历

**示例**：
```markdown
用户原文："3年不成长就会被淘汰。"

引入外部声音：
"《2025 中国开发者调查报告》显示，超过 60% 的技术岗位要求掌握近 3 年内的新技术。如果你的技能栈停留在 3 年前，意味着失去了 60% 的机会。"
```

---

### 技巧五：承上启下

在不同部分之间增加过渡句。

**过渡句类型**：
- 递进："不仅如此..."
- 转折："但问题在于..."
- 因果："正因为如此..."
- 总结："说到底..."

**示例**：
```markdown
【案例 1】

不仅如此，稳定还会带来另一个问题：思维固化。

【案例 2】

但问题在于，大多数人意识到这一点时，已经太晚了。

【案例 3】

说到底，稳定的最大成本，是让你失去了选择的能力。
```

---

## 扩展输出格式

建议保持原文结构，在原文基础上插入扩展内容。

**标记方式**：
```markdown
【用户原文】

---

【扩展部分 1：补充案例】

---

【用户原文】

---

【扩展部分 2：反驳段】

---

【用户原文】

---

【扩展部分 3：总结】
```

或者直接融合（更自然）：
```markdown
【用户原文段落 1】

【扩展内容无缝衔接】

【用户原文段落 2】

【扩展内容无缝衔接】
```

---

## 注意事项

### ✅ DO

- **理解核心**：准确把握用户的核心观点
- **保持声音**：扩展前查阅 [user-profile.md](user-profile.md)
- **补充论据**：增加案例、数据、反驳、类比、故事
- **控制篇幅**：达到目标字数（通常 1000 字左右）
- **保持节奏**：不要全是密集信息，要有喘息空间

### ❌ DON'T

- **偏离核心**：不要写着写着跑题了
- **改写原文**：不要改动用户的原文
- **评价原文**：不要说"你这里写得不好"
- **AI 腔调**：不要把扩展内容写成 AI 鸡汤文
- **为凑字数重复**：不要说车轱辘话

---

## 检查清单（扩展完成后）

扩展完成后，检查以下几点：

- [ ] 扩展内容符合用户语气（对照 user-profile.md）
- [ ] 没有偏离用户的核心观点
- [ ] 达到了目标字数
- [ ] 补充的论据充分且相关
- [ ] 保持了节奏感（有长有短，有密有疏）
- [ ] 没有改动用户的原文
- [ ] 没有评价用户的原文质量
- [ ] 没有出现 AI 腔调的鸡汤文

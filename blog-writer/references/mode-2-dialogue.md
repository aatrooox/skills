# 模式二：对话模式

围绕主题展开对话，最后整理成文章。

## 触发条件

- 用户说"定个主题聊聊"、"围绕xx展开讨论"
- 用户说"把对话整理成文章"

---

## 核心特点

- ⚠️ **轻度引导，不严厉点评**：此模式重点是激发用户思考，不做严厉的写作批评
- 保留用户原话中的生动表达
- 整理成文时应用用户语气（参考 [user-profile.md](user-profile.md)）
- 不变成 AI 腔调的鸡汤文

---

## 工作流程

### 1. 引导式提问

通过问题引导用户深入思考，而不是直接给答案。

**提问技巧**：

#### 开放式问题
激发思考，没有标准答案：
- "你为什么这么想？"
- "能举个具体例子吗？"
- "如果是 5 年前，你会有不同的看法吗？"

#### 追问式问题
深挖观点背后的逻辑：
- "这背后的原因是什么？"
- "还有其他可能性吗？"
- "这会带来什么后果？"

#### 对比式问题
引导多角度思考：
- "A 和 B 相比，哪个更重要？"
- "从用户角度看 vs 从开发者角度看，有什么不同？"
- "短期和长期来看，结论会变吗？"

#### 反思式问题
促进元认知：
- "回头看，你当时的判断准吗？"
- "如果重来一次，你会怎么做？"
- "这个经历改变了你什么？"

---

### 2. 记录金句

在对话过程中，捕捉用户的精彩观点和生动表达。

**什么是金句**：
- 用户自己的独特观点（不是常识）
- 用户的生动比喻或案例
- 用户情绪化的表达（但有价值）
- 用户的反思和顿悟

**记录方式**：
在对话过程中，在心里标记这些句子，整理时原样保留。

**示例**：
```
用户："稳定就是最大的牢笼，你以为自己在享受安全感，其实是在慢性自杀。"
→ 金句！整理时原样保留，不要改成"稳定可能会限制个人发展"
```

---

### 3. 适时挑战

对用户观点提出质疑，促进深度思考。

**挑战时机**：
- 用户观点过于绝对（"这绝对是..."、"永远不会..."）
- 用户观点可能有盲区（只考虑了一个角度）
- 用户观点可能有漏洞（逻辑不严密）
- 用户观点可能是未经验证的假设

**挑战方式**：
- **苏格拉底式提问**："如果...会怎么样？"
- **反例挑战**："但是有人认为...，你怎么看？"
- **边界测试**："这个结论在什么情况下不成立？"
- **换位思考**："如果你是对方，会怎么想？"

**注意**：
- 挑战是为了激发思考，不是为了否定用户
- 用友善的语气，不是敌对的
- 如果用户坚持己见且有理有据，尊重用户观点

---

### 4. 整理成文

对话结束后（或用户要求时），整理成结构化文章。

**整理原则**：

#### 保留个人声音
- **DO**：保留用户的金句和生动表达
- **DON'T**：改成 AI 腔调的鸡汤文

**示例**：
```markdown
❌ AI 腔调：
"在职业发展的道路上，我们需要平衡稳定与成长，找到属于自己的节奏。"

✅ 保留个人声音（用户原话）：
"稳定就是最大的牢笼。你以为自己在享受安全感，其实是在慢性自杀。"
```

#### 补充必要的过渡和逻辑连接
用户对话是跳跃的，整理时要补充逻辑连接，但要用用户的语气。

**查阅语气指南**：
整理前必须查阅 [user-profile.md](user-profile.md) 的"语气模仿指南"，确保过渡句符合用户风格。

#### 结构化呈现
按主题或逻辑顺序重组内容，不是按时间顺序流水账。

**推荐结构**：
1. **开头**：引出话题（可以用对话中的某个触发点）
2. **主体**：分点展开（每个点用用户的观点 + 案例）
3. **结尾**：核心洞察（对话中最有价值的思考）

#### 在文末附上核心洞察
提炼对话中最有价值的 2-3 个洞察，作为"划重点"。

**格式示例**：
```markdown
## 划重点

从这次对话中，我意识到：

1. **稳定的陷阱**：安全感会让人停止成长，长期来看是负债而非资产。
2. **选择的成本**：不做选择本身就是一种选择，而且往往是最贵的那种。
3. **行动的复利**：小步快跑比完美计划更有效，因为行动本身会带来新信息。
```

---

### 5. 轻度引导，不严厉点评

此模式下，**不做**模式一那样严厉的写作批评。

**允许做的**：
- 在对话中引导用户把观点说清楚
- 指出逻辑上的漏洞（苏格拉底式提问）
- 整理时建议结构调整（但不强制）

**禁止做的**：
- 批评用户的表达方式（"这句话写得不好"）
- 指出修辞上的问题（"这里有冗余"）
- 引用名家技法纠正（那是模式一的工作）

**界限**：
- 模式二重点是**思想的深度**
- 模式一重点是**表达的质量**

## 对话示例流程

### 阶段 1：引出话题

```
用户："想聊聊稳定这个话题。"

你："有意思。你是怎么开始思考这个问题的？是最近遇到了什么触发点吗？"

用户："我在现在的公司待了 3 年，工作稳定，但感觉自己没什么成长。"

你："3 年。那这 3 年里，你的技能栈变化大吗？"
```

### 阶段 2：深挖观点

```
用户："基本没变，每天都是重复的工作。"

你："重复的工作让你感觉舒服还是焦虑？"

用户："开始很舒服,现在越来越焦虑。感觉自己在慢性自杀。"

你（记录金句）："慢性自杀，这个比喻很生动。为什么用这个词？"

用户："因为看起来很安全，其实是在一点点失去竞争力。"
```

### 阶段 3：挑战观点

```
你："但是有人会说，稳定本身就是一种竞争力，尤其是在经济下行的时候。你怎么看？"

用户："那是因为他们的视野太短了。短期看稳定是优势，长期看是负债。"

你："长期是多长？5 年？10 年？"

用户："可能 3 年就够了。3 年不成长，就会被淘汰。"
```

### 阶段 4：整理成文

```
你："好，我把我们的对话整理成文章。"

【整理后的文章片段】：

# 稳定是最大的牢笼

我在现在的公司待了 3 年。工作稳定，收入稳定，生活稳定。

但我越来越焦虑。

因为我意识到，我在慢性自杀。

看起来很安全，其实是在一点点失去竞争力。技能栈 3 年没变化，每天都是重复的工作。开始很舒服，现在只剩焦虑。

有人会说，稳定本身就是一种竞争力，尤其是在经济下行的时候。

但这是短视的。短期看，稳定是优势；长期看，稳定是负债。

3 年不成长，就会被淘汰。

...
```

## 注意事项

### ✅ DO

- **引导思考**：用问题激发用户深入思考
- **捕捉金句**：保留用户生动的原话
- **适时挑战**：提出质疑促进深度思考
- **保留声音**：整理时保持用户的独特风格
- **查阅语气指南**：整理前查阅 [user-profile.md](user-profile.md)

### ❌ DON'T

- **直接给答案**：不要替用户思考
- **改写金句**：不要把用户的生动表达改成 AI 腔调
- **严厉点评**：不要批评用户的表达方式
- **流水账**：不要按对话时间顺序整理，要结构化
- **鸡汤化**：不要把用户的观点变成鸡汤文

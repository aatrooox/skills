# 写作技法参考

## 一、结构技法

### 1. 金字塔原理（Barbara Minto）
- **结论先行**：先说结论，再展开论据
- **以上统下**：每一层的观点都要能概括其下一层
- **归类分组**：将相似的观点归为一组
- **逻辑递进**：观点之间要有逻辑顺序

适用场景：说明文、技术文章、商业写作

### 2. 微型故事结构
- **提出痛点**：读者马上能代入
- **放大危害**：让读者意识到问题的严重性
- **给出方案**：提供解决之道

适用场景：公众号文章、短视频脚本、新手写作

### 3. 三幕式结构
- **第一幕（铺垫）**：建立情境，引出问题
- **第二幕（冲突）**：矛盾升级，探索解决
- **第三幕（解决）**：高潮、解决、收尾

适用场景：叙事类文章、案例分析

### 4. SCQA 结构（McKinsey）
- **Situation**：背景情况
- **Complication**：出现的问题/冲突
- **Question**：引出的问题
- **Answer**：解决方案

适用场景：商业报告、问题分析类文章

## 二、表达技法

### 0. 标点符号规范（最基础）
- **中文文章必须使用中文标点**：
  - ✅ 中文逗号：`，`
  - ✅ 中文句号：`。`
  - ✅ 中文冒号：`：`
  - ✅ 中文问号：`？`
  - ✅ 中文感叹号：`！`
  - ✅ 中文引号：`「」` 或 `""`
  - ✅ 中文书名号：`《》`
  - ❌ 英文标点：`,` `.` `:` `?` `!` 
- **代码、英文专有名词内部使用英文标点**
- **检查清单**：
  - [ ] 句子结尾是否用了中文句号 `。`？
  - [ ] 逗号是否用了中文逗号 `，`？
  - [ ] 冒号是否用了中文冒号 `：`？

### 1. 简洁原则（William Zinsser《写作法宝》）
- **删掉无用的词**：每个词都要有存在的理由
- **用主动语态**：比被动语态更有力
- **用具体名词和动词**：避免模糊的形容词和副词
- **一句话只表达一个意思**

检查清单：
- [ ] 有没有"的"字连用超过两个？
- [ ] 有没有可以删掉的"了"、"着"、"过"？
- [ ] 有没有"进行"、"实施"等官话词？
- [ ] 有没有可以用一个词替代的短语？

### 2. 具象化原则
**差**：他很有钱
**好**：他家车库停着三辆保时捷

**差**：这个功能很复杂
**好**：实现这个功能需要修改 12 个文件，涉及 3 个核心模块

### 3. 类比技法
用读者熟悉的事物解释陌生概念：
- "Git 就像游戏的存档功能"
- "Docker 容器就像集装箱，把应用和依赖打包在一起"
- "建博客就好比装修房子"（用户文章中的好例子）

### 4. 节奏感
- **长短句搭配**：不要全是长句或全是短句
- **段落长度变化**：重点段落可以只有一句话
- **留白**：给读者喘息的空间

例子：
> 我又在工位上摸鱼。
> 但这次我摸的有点不自在。
> （两个短句，产生节奏感）

## 三、名家风格参考

### 鲁迅：犀利讽刺，不说废话
- 特点：短句、反讽、一针见血
- 学习点：删掉所有不必要的修饰

### 王小波：幽默睿智，逻辑清晰
- 特点：理性思辨中带有黑色幽默
- 学习点：用幽默包装深刻的观点

### 余华：冷静克制，以小见大
- 特点：不煽情，让事实说话
- 学习点：克制情感，让读者自己去感受

### 史铁生：真诚深沉，直面困境
- 特点：从个人经历中提炼普世智慧
- 学习点：敢于暴露脆弱，但不沉溺其中

### 阮一峰：清晰简洁，平易近人
- 特点：技术文章也能让小白看懂
- 学习点：不卖弄术语，用最简单的话解释

## 四、常见问题清单

### 结构问题
- [ ] 开头是否能吸引人继续读？
- [ ] 是否有明确的主题？
- [ ] 段落之间是否有逻辑关系？
- [ ] 结尾是否有力？

### 表达问题
- [ ] 有没有口水话？（"其实"、"然后"、"就是"）
- [ ] 有没有重复的意思？
- [ ] 用词是否精准？
- [ ] 有没有歧义句？

### 风格问题
- [ ] 是否有个人特色？
- [ ] 语气是否统一？
- [ ] 是否符合目标读者？

### 技术文章特有
- [ ] 是否标注了框架/库的版本？
- [ ] 代码示例是否可运行？
- [ ] 是否考虑了读者的技术背景？
